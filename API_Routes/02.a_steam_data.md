
# 🎮 Steam Games API Documentation


---

## 🛠 1. Basic CRUD Operations 

*Standard operations for single-resource management.*

1. `GET /api/v1/games` - Fetch all games (default limit 50).
2. `GET /api/v1/games/:appid` - Get details of a single game.
3. `POST /api/v1/games` - Create a new game entry.
4. `PUT /api/v1/games/:appid` - Replace an existing game record.
5. `PATCH /api/v1/games/:appid` - Update specific fields (e.g., update just the price).
6. `DELETE /api/v1/games/:appid` - Remove a game from the database.
7. `GET /api/v1/games/:appid/exists` - Quick check (Boolean) if a game exists.

---

## 🔍 2. Advanced Filtering & Searching 

*Practice string manipulation and multiple query parameter handling.*

8. `GET /api/v1/games/search?q=query` - Partial name search (Case-insensitive).
9. `GET /api/v1/games/filter?genre=Action&year=2024` - Filter by multiple exact fields.
10. `GET /api/v1/games/filter/price?min=0&max=15` - Filter by price range.
11. `GET /api/v1/games/filter/recommendations?min=500` - High-performing games.
12. `GET /api/v1/games/filter/date?start=2023-01-01&end=2023-12-31` - Date range filtering.
13. `GET /api/v1/games/genre/:genreName` - Direct route for specific genre categories.
14. `GET /api/v1/games/developer/:devName` - Filter by developer.
15. `GET /api/v1/games/publisher/:pubName` - Filter by publisher.
16. `GET /api/v1/games/tags/multi?tags=Indie,RPG` - Filter games containing *all* specified tags.
17. `GET /api/v1/games/free-to-play` - Shortcut for games with price `0.00`.
18. `GET /api/v1/games/early-access` - Filter games currently in Early Access.
19. `GET /api/v1/games/vr-only` - Filter for Virtual Reality exclusive titles.

---

## 📦 3. Pagination & Sorting 

*Essential for performance with large datasets.*

20. `GET /api/v1/games/paged?page=1&limit=20` - Standard page-based pagination.
21. `GET /api/v1/games/offset?skip=100&take=50` - Offset-based pagination.
22. `GET /api/v1/games/sort/price-asc` - Sort by lowest price first.
23. `GET /api/v1/games/sort/price-desc` - Sort by highest price first.
24. `GET /api/v1/games/sort/newest` - Sort by most recent release date.
25. `GET /api/v1/games/sort/popularity` - Sort by recommendation count.
26. `GET /api/v1/games/sort/alphabetical` - Sort by name (A-Z).
27. `GET /api/v1/games/next/:appid` - Get the "next" game in the list for infinite scrolling.

## 📈 4. Data Analytics & Aggregation 

*Calculated routes that require processing the whole dataset.*

28. `GET /api/v1/stats/total` - Count total games in the dataset.
29. `GET /api/v1/stats/average-price` - Calculate mean price across all games.
30. `GET /api/v1/stats/years-distribution` - Count of games per release year.
31. `GET /api/v1/stats/genres/top` - List top 10 genres by game count.
32. `GET /api/v1/stats/developers/top` - List top 10 developers by game count.
33. `GET /api/v1/stats/revenue-estimate` - (Practice) Mock revenue by multiplying price * recommendations.
34. `GET /api/v1/stats/category-breakdown` - Percentage of single-player vs multi-player games.
35. `GET /api/v1/stats/price-tier` - Count of games in "Budget", "Mid", and "Premium" tiers.
36. `GET /api/v1/stats/most-recommended` - The single game with the highest user feedback.

## ✨ "Good to Have" (Professional Enhancements)

Implementing these will elevate your project from a basic exercise to a professional portfolio piece:

* **Middleware Logging:** Every request should be logged to the console (Method, URL, Time taken).
* **Response Wrapping:** Standardize all responses like this:
```json
{
  "success": true,
  "data": [],
  "timestamp": "2026-05-11T..."
}

```


* **Virtual Fields:** In your code, create a virtual field called `is_free` that returns `true` if price is `0`.
* **Search Debouncing:** Handle high-frequency search requests by ensuring the server doesn't choke on partial queries.
* **Request Sanitization:** Automatically trim whitespace and escape special characters in user-submitted text.
